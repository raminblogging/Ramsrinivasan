Data Sync
Connect admin data to your main site

📤 Export — Download site-data.json

Downloads all your content as a site-data.json file. Commit this file to the root of your GitHub repo so your main index.html can fetch and display it automatically.


      


      

        
📥 Import — Restore from JSON

        
Upload a previously exported site-data.json to restore your data on a new browser or device. Warning: This overwrites all current data.

        

          
          
        

      


      

        
🗄️ Current Data in Browser

        
These values are stored in your browser's localStorage and are read by your main site automatically when both files are on the same domain.

        

      


      

        
📋 How to Connect to Your Main Site

        

          
1
Same domain (automatic): If you upload both admin.html and index.html to the same GitHub Pages repo at ramsrinivasan.in, the blog section on your main site already reads your blog posts from localStorage automatically — no extra setup needed.

          
2
JSON file (manual): Click "Download site-data.json" above, then commit that file to your GitHub repo root. Your updated index.html fetches this file as a fallback when localStorage is empty (e.g. for first-time visitors).

          
3
After writing a new post: Publish it in the Blogs section, then export a fresh site-data.json and commit it. Your live site will show the new post instantly for all visitors.
